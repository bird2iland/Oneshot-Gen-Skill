# 视频生成 Skill 示例对话文档

本文档提供多个完整的使用场景示例，展示视频生成 Skill 在不同情况下的交互流程。

---

## 示例 1：标准流程（预设选择 + 图片上传 + 生成）

### 场景描述
用户希望生成一个城市延时视频，按照标准流程完成从预设选择到视频生成的全过程。

### 完整对话

```
用户：帮我生成一个城市延时视频

Agent: [触发识别] 检测到关键词"生成视频"、"延时"
       [调用 detect_intent]
       
Agent: [步骤1 - 环境检查] 调用 check_status()
       ✓ 即梦 CLI 已安装并登录
       
Agent: [步骤2 - 列出预设] 调用 list_presets()
       
       可用的风格预设：
       1. realistic - 写实风格
       2. cinematic - 电影质感
       3. anime - 动漫风格
       
       可用的主题预设：
       4. timelapse - 延时摄影
       5. slowmo - 慢动作
       
       可用的运镜预设：
       6. shuttle - 穿梭机视角
       7. drone - 航拍视角
       
       请选择预设组合（输入编号或名称，用 + 连接）

用户：realistic + timelapse + shuttle

Agent: [步骤2 - 确认预设组合]
       已选择预设组合：
       - realistic: 写实风格
       - timelapse: 延时摄影
       - shuttle: 穿梭机视角

Agent: [步骤3 - 询问参考图]
       请上传 1-4 张参考图片（可选）
       或描述你想要的视频内容

用户：[上传2张图片] 城市从清晨到黄昏

Agent: [步骤4 - 优化提示词] 调用 optimize_prompt()
       图片分析：城市景观，日出到日落
       
       优化后的提示词：
       ┌────────────────────────────────────────┐
       │ 写实风格城市延时摄影，穿梭机视角，     │
       │ 展现城市从清晨第一缕阳光到黄昏华灯    │
       │ 初上的壮丽变迁，高楼大厦的光影变化，  │
       │ 街道人流车流穿梭不息，天空云层流动，  │
       │ 色彩从温暖的橙红过渡到冷调的蓝紫。    │
       │ 风格：高画质、电影级调色、稳定运镜。  │
       └────────────────────────────────────────┘
       
       满意这个提示词吗？（可以提出修改意见）

用户：可以

Agent: [步骤5 - 询问导出或生成]
       请选择下一步：
       1. 直接生成视频
       2. 导出提示词到文件

用户：直接生成

Agent: [步骤6 - 执行生成] 调用 generate_video()
       正在生成视频...
       提示词已提交到即梦平台
       
Agent: [步骤7 - 返回结果]
       ✓ 视频生成完成！
       
       文件信息：
       - 路径: output/videos/city_timelapse_20260422.mp4
       - 时长: 5秒
       - 分辨率: 1920x1080
       
       [视频预览]
```

### 关键步骤标注

| 步骤 | 动作 | 工具函数 | 说明 |
|------|------|----------|------|
| 1 | 环境检查 | `check_status()` | 验证 CLI 安装和登录状态 |
| 2 | 列出预设 | `list_presets()` | 展示所有可用预设供用户选择 |
| 3 | 询问参考图 | - | 收集用户上传的图片或描述 |
| 4 | 优化提示词 | `optimize_prompt()` | 基于预设和图片生成优化提示词 |
| 5 | 询问下一步 | - | 让用户选择导出或生成 |
| 6 | 执行生成 | `generate_video()` | 调用 CLI 生成视频 |
| 7 | 返回结果 | - | 展示生成结果和文件信息 |

### 工具函数调用说明

```
check_status() → 返回 {installed: true, logged_in: true}
list_presets() → 返回预设列表 JSON
optimize_prompt(presets, images, description) → 返回优化后提示词
generate_video(prompt) → 返回视频文件路径
```

```
export_prompt(prompt, filename) → 返回文件路径
```

## 示例 4：错误处理场景（环境未配置）

### 场景描述
用户首次使用视频生成功能，但环境尚未配置，Agent 检测到问题并提供安装指引。

### 完整对话

```

       [调用 detect_intent]

Agent: [步骤1 - 环境检查] 调用 check_status()
       ✗ 即梦 CLI 未安装

Agent: [错误处理 - 提供安装指引]
       ┌────────────────────────────────────────┐
       │ 检测到即梦 CLI 未安装                  │
       │                                        │
       │ 请按以下步骤完成配置：                 │
       │                                        │
       │ 步骤 1: 安装即梦 CLI                   │
       │   curl -fsSL https://jimeng.jianying.com/cli | bash
       │                                        │
       │ 步骤 2: 登录账户                       │
       │   dreamina login                       │
       │                                        │
       │ 步骤 3: 验证安装                       │
       │   dreamina --version                   │
       │                                        │
       │ 完成配置后，请重新说"生成视频"来触发  │
       │ 此功能。                               │
       └────────────────────────────────────────┘
       
       需要我详细解释某个步骤吗？

用户：第二步登录怎么操作？

Agent: [详细指导]
       dreamina login 命令会打开浏览器进行授权：
       
       1. 终端会显示一个授权链接
       2. 点击链接或复制到浏览器打开
       3. 使用字节跳动账号登录
       4. 授权成功后，终端会自动完成登录

       - dreamina login --web    # 强制使用网页登录
```

| 步骤 | 动作 | 工具函数 | 说明 |
|------|------|----------|------|
| 1 | 环境检查 | `check_status()` | 返回安装状态为 false |
| - | 错误处理 | - | 提供清晰的安装指引 |
| - | 交互支持 | - | 回答用户的配置问题 |

### 工具函数调用说明

```
check_status() → 返回 {installed: false, logged_in: false, error: "CLI not found"}
// 当检测到未安装时，Agent 应停止后续流程
// 提供安装指引，等待用户完成配置
```

```
recommend_presets(keyword, preferences={})

{
  "recommended": ["realistic", "timelapse", "drone"],
```
